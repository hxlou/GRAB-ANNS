# lightCagra

一个简化版本的cagra代码，方便后续进一步修改

依赖`faiss`库，可以通过以下指令安装：

```bash
conda create -n lightCagra
conda activate lightCagra
conda install conda-forge::faiss-gpu        # 包含所有必须环境
```

构建方法

```bash
mkdir build & cd build
cmake ..
make -j10
```


### 方案2 remote edge

1. 数据全部放在一起，只在初始化的时候需要数据是按顺序排布好的
2. 在 CPU 侧维护一个 time stamp -> idx 的映射
    1. 在指定范围进行查询时，从指定的time stamp生成随机种子
    2. 在全部范围进行查询时，在全部范围生成随机种子即可
3. 对于邻居列表中每个点的邻居
    1. 前 28 用于局部连接，后 4 用于全局连接
    2. device 端需要维护一个反查表，用于快速定位不同的邻居位于哪个time stamp

构建方案：
- 全量构建每个子图
- 针对每个子图，我们重新更新其最后 4 个元素
    - 最简单的策略是对于桶 x 中的每个元素，都去桶外的检索离自己最近的几个节点，并更新自己最后的 4 个邻居信息
    - 构建一个全量 IVF-PQ 索引，来近似找到最临近的几个图
    - 最大限度提升图的连通性
        - 尽量保证每个点的邻居节点的remote edge和自己的remote edge不重叠

更新方案：
- 写一个特殊的查询方法，同时记录内部的点和外部的点，直接找到我指向谁
- 更新谁指向我
    - 前 28 个按照旧的策略直接来进行
    - 后 4 个   再说吧，还没写到呢
